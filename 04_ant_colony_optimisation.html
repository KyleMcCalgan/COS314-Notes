Ant Colony Optimisation

The ACO construction loop, probability rule, pheromone update equation, and the parameters that control exploration and exploitation.

Practical View

ACO is used for constructive combinatorial optimisation. Ants build solutions step by step, and pheromone values bias future ants toward stronger components of past solutions.

Typical space: graph or construction graph
Key memory: pheromone on components
Main balance: alpha and beta

Algorithm

1
Initialise parameters and pheromone values.

Set initial tau_ij values and decide how many ants will be used.

2
Each ant constructs a solution.

At every step it chooses the next feasible component using pheromone and heuristic information.

3
Optionally apply local search.

The notes call these daemon actions.

4
Update pheromone trails.

Evaporate old pheromone and deposit new pheromone on strong solution components.

5
Repeat until termination.

Common criteria are iteration count or time budget.

Probability Rule

p(i,j) = (tau_ij^alpha * eta_ij^beta) / sum(tau_il^alpha * eta_il^beta)
When You Use It Sub-topic 3. Use this every time an ant must choose its next city or next feasible component during solution construction.
P(edge i -> j) = (pheromone^pheromone_weight * desirability^heuristic_weight) / sum over all valid edges(same)

The denominator just normalises the values so that all valid next-edge probabilities add up to 1.

SymbolMeaning
tau_ijPheromone on edge or component (i,j)
eta_ijHeuristic desirability, usually 1 / distance
alphaPheromone weight
betaHeuristic weight
N(s)Feasible neighbourhood from the current partial solution
Interpretation
  • Higher alpha means stronger reliance on pheromone trails.
  • Higher beta means stronger reliance on heuristic greediness, such as short edges.
  • Together they control exploration versus exploitation.

Pheromone Update

tau_ij(t + 1) = (1 - rho) * tau_ij(t) + Delta tau_ij
When You Use It Sub-topic 4. Apply the pheromone update once per iteration, after all ants have completed their tours for that iteration.
new_pheromone(i,j) = (1 - evap_rate) * old_pheromone(i,j) + total_deposited(i,j)
Delta tau_ij = sum(Delta tau_ij^k)

The first term keeps some memory of the old trail, while the second term adds all pheromone deposits made by ants that used that edge.

Deposit Rule

Delta tau_ij^k = Q / L_k if ant k used edge (i,j), otherwise 0
When You Use It Also in Sub-topic 4. Compute this per ant first, then sum all ants' deposits into the main pheromone update formula.
deposit(i,j) = pheromone_constant / tour_length if ant used this edge, else 0

Shorter tours get a larger Q / L_k value, so better ants reinforce their edges more strongly. If an ant did not walk on the edge, it deposits nothing there.

ParameterMeaning
rhoEvaporation rate, with 0 < rho < 1
QPheromone scaling constant
L_kLength or cost of ant k's full solution
Shorter tours receive larger pheromone deposits because Q / L_k is bigger when L_k is smaller.

Heuristic Value

eta_ij = 1 / d_ij
When You Use It Sub-topic 3. This is pre-computed before the algorithm starts, then fed into the probability rule as the desirability of each edge.
desirability(i,j) = 1 / distance(i,j)

Short edges get high desirability, long edges get low desirability, and this heuristic value does not change during the run.

Worked Selection Example

Suppose an ant at node A can choose AB or AC, with alpha = 1 and beta = 2.

PathtauetaNumerator
AB40.14^1 * 0.1^2 = 0.04
AC20.52^1 * 0.5^2 = 0.50
Denominator = 0.04 + 0.50 = 0.54
P(AB) = 0.04 / 0.54 = 7.4%, P(AC) = 0.50 / 0.54 = 92.6%

Even though AB has more pheromone, AC is far more likely because the heuristic term is much stronger here.

Parameters You Need To Know

ParameterRole
alphaPheromone weight. High alpha means ants follow existing trails more strongly, so exploitation increases.
betaHeuristic weight. High beta makes ants behave more greedily by preferring shorter edges more strongly.
rhoEvaporation rate, between 0 and 1. High rho means pheromone fades faster, so the colony has less memory and explores more.
QPheromone constant. Larger Q means more pheromone is deposited per tour.
tauPheromone level on an edge. It is updated every iteration and usually starts equal on all edges at some initial tau_0.
etaHeuristic desirability, usually 1 / distance. It is fixed and never updated.
mNumber of ants. Often chosen equal to the number of cities.
Local search strategyOptional post-processing step for complete solutions
Termination criterionMaximum iterations, time, or convergence threshold

Tutorial-Style Route Choice Example

Suppose an ant is at Node 1 and may move to Node 2, Node 3, or Node 4. Let alpha = 1, beta = 2, and eta_ij = 1 / C_ij.

PathCostPheromoneHeuristicNumerator
1 -> 2240.54 * 0.5^2 = 1.0
1 -> 35100.210 * 0.2^2 = 0.4
1 -> 4111.01 * 1.0^2 = 1.0
Total = 1.0 + 0.4 + 1.0 = 2.4
P(1 -> 2) = 1.0 / 2.4 = 0.4167, P(1 -> 3) = 0.4 / 2.4 = 0.1667, P(1 -> 4) = 1.0 / 2.4 = 0.4167

This is a good example of ACO balancing two signals at once: path 1 -> 3 has the most pheromone, but its cost is worse, so it does not dominate.

What The Algorithm Is Really Doing

During construction Each ant is building a candidate solution incrementally. At this stage, the algorithm is biased but still probabilistic.
During update The colony is learning which edges or components tend to appear in strong solutions, while evaporation prevents old information from dominating forever.
