Complete GE reference table covering all key concepts keywords formulas and processes

Term
Definition
Grammatical Evolution
An evolutionary algorithm that searches the program space using variable-length binary string chromosomes decoded into programs via a BNF grammar. It extends the GP idea by separating genotype and phenotype.
Genotype
The binary string chromosome that gets evolved. Crossover and mutation operate here. It is not directly evaluated for fitness.
Phenotype
The executable program produced after decoding the genotype through the grammar. This is what gets evaluated against the fitness cases.
Codon
One 8-bit chunk of the chromosome. Convert it to decimal, then use the mapping equation to select a grammar rule. Each codon answers one grammar decision.
Codon value range
0 to 255. Modulo maps any value to a valid rule index no matter how many rules the current non-terminal has.
Variable-length string
Chromosomes in GE do not all have the same length. Each individual can contain a different number of codons.
Program space
The search space GE explores: all programs expressible by the grammar.
Derivation
The process of converting genotype to phenotype by consuming codons and expanding non-terminals.
Wrapping
If codons run out before derivation is complete, mapping loops back to the start of the codon sequence. If the phenotype is still incomplete after the allowed wraps, the individual is invalid.
Invalid individual
A chromosome that fails to produce a complete phenotype. It is usually assigned the worst fitness so that selection removes it.
Fitness function
Problem-dependent scoring function applied to the phenotype. In symbolic regression this is often SAE.
SAE
SAE = sum |program(x_i) - y_i|. Lower is better, and 0 means a perfect fit.
Fitness cases
The training input/output pairs used to evaluate each phenotype.
Deterministic mapping
With the same grammar and codon sequence, GE always produces the same phenotype.
Term
Definition
CFG
Context-Free Grammar. GE uses a CFG written in BNF notation.
BNF
Backus-Naur Form. It uses ::= for "can be replaced by" and | for alternatives.
G = <N, T, P, S>
Formal grammar definition: N = non-terminals, T = terminals, P = production rules, S = start symbol.
Non-terminals
Symbols like <expr> and <op> that still need to be expanded.
Terminals
Final output symbols like +, x, or 3. They do not expand further.
Production rules
The possible expansions for each non-terminal. Count the alternatives to know how many rules there are.
Start symbol
The first non-terminal to be expanded in every derivation.
::=
Separates the non-terminal from its right-hand-side expansions.
|
Separates alternative rules. The number of alternatives is the number of rules for that non-terminal.
Arity
The number of inputs an operator needs, for example binary operators like + or * take two.
Domain knowledge
The grammar itself can encode what counts as a valid or useful program for the problem.
LHS / RHS
Left-hand side is the non-terminal being defined. Right-hand side is the set of expansions it can become.
Term / step
Definition
Mapping equation
rule = codon_decimal % number_of_rules. This gives the zero-based rule index to apply.
Modulo (%)
Returns the remainder after division, guaranteeing a valid rule index.
Number of rules
Count the alternatives for the current non-terminal.
Rule index
Zero-based: rule 0 is the first option, rule 1 the second, and so on.
Leftmost derivation
Always expand the leftmost non-terminal first. This is what makes the process deterministic.
Multi-symbol expansion
A rule can expand into multiple symbols at once, for example <expr> <op> <expr>.
Derivation string
The current partially expanded expression, containing both terminals and non-terminals.
Codon pointer
Tracks which codon is used next. It advances after each expansion and wraps if needed.
Binary to decimal
Convert each 8-bit codon to decimal before applying the mapping equation.
Derivation tree
A tree showing how the start symbol expands into the final phenotype.
Three questions per step
1. Which non-terminal am I expanding? 2. How many rules does it have? 3. What does codon % rules give me?
Component
Definition
Initial population
Randomly generated variable-length binary strings within user-defined bounds.
Selection
Usually tournament or fitness-proportionate selection, just like in GA.
Crossover
Single-point crossover on binary strings. Offspring lengths can differ from the parents.
Cut point rule
Choose the cut point from [1, shortest_parent_length - 1].
Variable-length offspring
Parents can have different lengths, so offspring lengths can vary too.
Mutation
Bit-flip mutation changes bits in the chromosome, which changes codon values and therefore grammar decisions.
Elitism
Best individuals are copied unchanged into the next generation.
Population replacement
Can be generational or steady-state, optionally with elitism.
Termination
Stop when the target fitness is reached or the maximum generation count is hit.
GE algorithm order
Generate population, decode, evaluate fitness, select parents, crossover, mutate, evaluate offspring, replace population, repeat.
What evolution changes
Only the chromosome changes. The grammar and fitness function stay fixed.
What fitness evaluates
Only the phenotype is scored. The chromosome is never evaluated directly.
