Genetic Programming

Program evolution using syntax trees, subtree crossover, tree mutation, and the key parameters that control GP runs.

What Makes GP Different?

Genetic Programming is an evolutionary algorithm like a GA, but the individuals are executable programs rather than fixed simple strings. In the notes, those programs are represented as syntax trees.

GA

Chromosomes represent candidate solutions directly.

Operators work on strings or arrays.

GP

Trees represent candidate programs or expressions.

Operators work on subtrees and terminals.

Algorithm Flow

1
Define function set and terminal set.

These determine what a valid program tree can contain.

2
Generate an initial population of trees.

Common methods are Full, Grow, and Ramped Half-and-Half.

3
Evaluate fitness.

Run or interpret each tree against the fitness cases.

4
Select parents.

The same scoped methods apply here too: tournament, fitness proportionate, and rank selection.

5
Apply genetic operators.

Subtree crossover is primary; grow or shrink mutation introduces diversity.

6
Replace population and check termination.

Use generational or steady-state replacement and optionally elitism.

Representation: Syntax Trees

Function set Internal nodes such as +, -, *, /, or domain-specific operators.
Terminal set Leaf nodes such as variables, constants, or zero-argument symbols.
/ / \ * - / \ / \ a c a b

The tree above represents the expression (a * c) / (a - b).

Initial Population Methods

MethodHow it builds treesMain practical effect
FullInternal nodes stay functions until the maximum depth is reachedProduces bushy, regular trees
GrowCan place functions or terminals before the maximum depthProduces irregular tree shapes
Ramped Half-and-HalfMix of Full and Grow across several depthsImproves initial diversity

Subtree Crossover

Definition Select one crossover point in each parent tree and swap the subtrees rooted at those points.
1
Select Parent 1 and Parent 2.

Use one of the scoped selection methods.

2
Pick one node in each tree.

Those nodes become the crossover points.

3
Swap the subtrees.

This creates two offspring trees.

4
Check depth limit.

If an offspring exceeds the maximum allowed depth, it must be pruned or rejected.

Apply crossover if r < p_c

Mutation

Mutation typeOperationEffect on tree size
Grow mutationReplace a terminal node with a randomly generated subtreeTree gets larger
Shrink mutationReplace a selected subtree with a terminal nodeTree gets smaller
GP notes emphasise depth limits during mutation as well. A mutation may be valid conceptually but still invalid if it breaks the allowed tree-depth constraint.

Worked Practical Pattern

If you get a GP question in the test, a strong answer usually follows this order:

  • Identify the function set and terminal set.
  • Show the candidate tree clearly.
  • State how fitness is computed from the training cases.
  • Show the selected crossover or mutation point.
  • Draw or describe the offspring tree after the operator.
  • Mention depth limits and whether elitism or replacement applies.

Subtree Crossover Demonstration

Parent 1
/ / \ * - / \ / \ a c a b
Parent 2
+ / \ x y

If the selected crossover fragment in Parent 1 is the * subtree and the selected fragment in Parent 2 is the entire + tree, the swap produces these offspring:

Offspring 1: (x + y) / (a - b) Offspring 2: a * c
This is the key GP intuition: crossover swaps pieces of executable structure, not just positions in a flat string.

Mutation Demonstration

MutationWhat you selectWhat you insertResult
Grow mutationA terminal leafA random subtreeTree becomes larger and potentially more expressive
Shrink mutationA subtreeA terminalTree becomes smaller and simpler

When doing these by hand, always identify the mutation point first, then rewrite only that part of the tree.

Exam Checklist

  • Explain the difference between GA strings and GP trees.
  • Identify function and terminal sets in a given problem.
  • Work subtree crossover by hand.
  • Distinguish grow mutation from shrink mutation.
  • State the role of tree-depth limits and why they matter.
