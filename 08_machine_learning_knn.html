Machine Learning and K-Nearest Neighbour

The supervised-learning basics, distance metrics, and KNN calculation pattern listed in the final exam scope.

Machine Learning Basics

Machine learning uses data to build a model or decision procedure that can make predictions on unseen cases. The important exam idea is generalisation: a model should perform well on new data, not only on the examples it saw during training.

Supervised learning Training examples include labels or target values. Classification predicts a category; regression predicts a numeric value.
Unsupervised learning Training data has no class labels. The algorithm discovers structure, such as clusters.
TermMeaning
InstanceOne row/example in the dataset.
Attribute or featureOne measured property of an instance.
Class labelThe target category in classification.
Training setData used to learn or store the classification behaviour.
Test setUnseen data used to estimate generalisation.

KNN Core Idea

K-nearest neighbour is a supervised method for classification or regression. It is instance-based: it stores the training instances and delays real work until a new query point must be classified.

KNN rule Find the k closest training instances to the new point. For classification, predict the majority class. For regression, predict the average target value.
No explicit model
Distance-based
Sensitive to scaling
Usually k = 3 or 5

KNN Algorithm

1
Prepare the data.

Normalise or scale features when different attributes use different units.

2
Choose k.

Small k is sensitive to noise. Larger k is smoother but may blur class boundaries.

3
Calculate distances.

Compute the distance from the query point to every training point.

4
Select the k closest neighbours.

Sort by distance and take the first k rows.

5
Predict.

Use majority voting for classification or averaging for regression.

Distance Metrics

Euclidean distance = sqrt(sum((p_i - q_i)^2))
Manhattan distance = sum(|p_i - q_i|)
MetricUseExample
EuclideanStraight-line distance in numeric feature space.sqrt((6 - 7)^2 + (4 - 2)^2) = sqrt(5) = 2.24
ManhattanAxis-by-axis distance, like city blocks.|12 - 4| + |9 - 2| = 15
If one feature has much larger numeric values than the others, it can dominate the distance. That is why normalisation is often needed before KNN.

Exam Traps

  • KNN does not train a compact mathematical model; it stores examples and compares at prediction time.
  • For k = 1, the nearest single example decides the class.
  • For k = 3 or k = 5, majority vote improves robustness against noisy examples.
  • For regression, do not vote. Average the target values of the nearest neighbours.
  • Always show the distance table before giving the class prediction.
