Heuristic Functions

Smart guesses that guide search toward the goal

What is a Heuristic?

A heuristic is a rule of thumb — a problem-specific piece of knowledge that helps guide a search towards a solution faster, without guaranteeing the optimal path.

Formal Definition A heuristic function h(n) is an evaluation function that estimates the cost from node n to the goal node. It uses domain knowledge to assess how "promising" a state is.

Without heuristics, we'd have to explore every possible state blindly (BFS/DFS). With heuristics, we can prioritise states that look more likely to lead to the goal — dramatically reducing search time.

Heuristic Value Scale

Heuristic values h(n) have specific meanings based on their magnitude:

h(n) = 0
h(n) small
h(n) large
h(n) = ∞
→ Goal state → Close to goal → Far from goal → Dead end (never explore)
Heuristic values are ALWAYS ≥ 0. A value of 0 means we ARE at the goal. Infinity means this node will never lead to the goal — skip it entirely.

Key Properties of Heuristics

Admissible Heuristic h(n) ≤ h*(n) for all nodes. The heuristic NEVER overestimates the actual cost to the goal. This is required for A* to guarantee an optimal solution.
Non-admissible Heuristic h(n) > h*(n) for some nodes. Overestimates the cost — may find a suboptimal path. Can still be useful but loses the optimality guarantee.
Good Heuristic Doesn't require long computation time. Closely approximates the true cost. More accurately informed = fewer nodes expanded.
Better Heuristic h₁ vs h₂ h₁ is better than h₂ if h₁ causes fewer nodes to be expanded during the search. More informed heuristics dominate.

Heuristic Functions for the 8-Puzzle

Three classic heuristics for the 8-puzzle problem — each increasingly informative:

Heuristic How it works Admissible? Notes
Tiles out of place Count how many tiles are NOT in their goal position ✅ Yes Each tile needs at least 1 move to reach its goal — never overestimates
Manhattan Distance For each tile, count how many squares it needs to move horizontally + vertically to reach its goal position. Sum all tiles. ✅ Yes More informative than tiles-out-of-place. Does not count diagonal moves or blocked paths.
Tile Reversals × 2 When two tiles need to swap positions directly, multiply count by 2 ✅ Yes Handles a special hard case — swaps require extra moves
Initial State: Goal State: ┌───┬───┬───┐ ┌───┬───┬───┐ │ 2 │ 8 │ 3 │ │ 1 │ 2 │ 3 │ ├───┼───┼───┤ ├───┼───┼───┤ │ 1 │ 6 │ 4 │ │ 8 │ │ 4 │ ├───┼───┼───┤ ├───┼───┼───┤ │ 7 │ │ 5 │ │ 7 │ 6 │ 5 │ └───┴───┴───┘ └───┴───┴───┘ Tiles out of place: {2, 8, 1, 6} = 4 tiles wrong (wait, let's check each) 1→wrong pos, 2→wrong pos, 8→wrong pos, 6→wrong pos = 4 tiles out of place (h=4) Manhattan Distance: Tile 2: needs to move 1 right → 1 Tile 8: needs to move 1 left, 1 up → 2 Tile 1: needs to move 1 up → 1 Tile 6: needs to move 1 down → 1 Total h = 5
Manhattan Distance is ALWAYS ≥ Tiles Out of Place. This means it expands fewer nodes. More informed = better. Both are admissible since neither overestimates real moves needed.

How Heuristics Are Defined

Heuristics are typically defined as a simplified/relaxed version of the problem. We pretend certain constraints don't apply, which lets us compute a lower bound on actual cost.

Example — 8-Puzzle relaxation Real puzzle: a tile can only move to an adjacent blank space.
Relaxed version for "tiles out of place": a tile can move to its goal position in a single step.
This relaxation makes h easy to compute, and since it ignores real constraints, it can only UNDERESTIMATE (= admissible).
Difficulty in practice Experience shows it is VERY DIFFICULT to devise good heuristics. Heuristics are fallible — they sometimes lead the search astray. No heuristic is perfect.

Why Heuristics Matter: Weak vs Strong Search

Search methods that use heuristics are called "weak" search methods — not because they're bad, but because the knowledge they incorporate is imprecise. It usually helps but doesn't always.

Without heuristics (BFS/DFS)
  • Explores blindly
  • Complete state space may be needed
  • Very slow for large spaces
  • Guaranteed to find goal (BFS)
With heuristics (Best-First, A*)
  • Guided exploration
  • Dramatically fewer nodes expanded
  • Much faster in practice
  • May miss optimal path (unless admissible)
