Genetic Algorithms

A practical walk-through of the GA cycle, operators, parameters, and worked binary examples.

Core Idea

A Genetic Algorithm is a population-based metaheuristic. Instead of improving one solution at a time, it maintains a whole population of candidate solutions and evolves them over generations.

Representation: chromosome
Search style: population based
Main operators: selection, crossover, mutation

Crossover

Main idea Crossover combines two parents to produce offspring. In GA notes it is the primary operator, so its application rate is usually high.
Apply crossover if r < p_c
Parent 1: 1 1 1 | 0 1 1 Parent 2: 1 0 1 | 0 1 0 ^ split point Offspring 1: 1 1 1 | 0 1 0 Offspring 2: 1 0 1 | 0 1 1
  • One-point crossover swaps the tails after one cut point.
  • Two-point crossover swaps the middle segment between two cuts.
  • Uniform crossover decides each gene position separately.

Mutation

Main idea Mutation introduces diversity and helps prevent premature convergence. It is secondary, so the rate is usually lower than crossover.
For each gene, mutate if r < p_m
Before: 1 1 1 0 1 0 ^ After: 1 1 1 1 1 0

For binary chromosomes, the standard mutation operator is bit-flip mutation: 0 -> 1 or 1 -> 0.

Replacement and Termination

ChoiceMeaningWhy it matters
Generational replacementEntire population is replaced by offspringSimple and clean generation boundaries
Steady-state replacementOnly some individuals are replacedLets good parents survive longer
ElitismBest individual is always preservedProtects the best-so-far solution from being lost
  • Typical termination conditions: maximum generations, threshold fitness, or no improvement for several generations.
  • GAs are stochastic and do not guarantee the global optimum.

How To Read A Full GA Run

Exploration comes from
  • Initial random population
  • Mutation
  • Weaker individuals still sometimes being selected
Exploitation comes from
  • Selection pressure
  • Crossover combining good building blocks
  • Elitism preserving the best-so-far solution
When you explain a GA, do not describe the operators in isolation. Always connect them to the bigger search behavior: which parts intensify the search, and which parts keep it diverse.

Exam Checklist

  • Explain why GA is population-based rather than single-solution search.
  • Write the full algorithm cycle in order.
  • Work one-point crossover and bit-flip mutation by hand.
  • State why p_c is usually high and p_m is lower.
  • Describe replacement strategies and elitism.
