Grammatical Evolution

Genotype to phenotype mapping, codons, modulo rule selection, and the full worked derivation pattern that is most likely to be tested practically.

Core Idea

Grammatical Evolution evolves binary genomes, then maps them through a grammar to produce executable structures or expressions. The big practical difference from GP is that GE evolves a genotype string first and only then derives the phenotype through grammar rules.

Algorithm Flow

1
Create an initial population of variable-length binary strings.

Each 8-bit block is a codon.

2
Map each genotype through a grammar.

Use codon values to choose production rules.

3
Evaluate fitness.

The phenotype is evaluated on the target problem.

4
Select, recombine, and mutate.

Selection works like other EAs. Single-point crossover and bit mutation are the main operators.

5
Replace population and check termination.

Repeat until a generation limit or objective condition is met.

Representation

Genotype Variable-length binary string made of codons.
Codon One 8-bit block in the genotype.
Phenotype The expression or program obtained after grammar-based derivation.
| 00000011 | 10100110 | 00000100 | 00011110 | 10101100 | 00000010 | ^ each 8-bit block is one codon

The Key Mapping Rule

rule index = codon decimal value % number of production rules
  • Expand the leftmost non-terminal first.
  • Consume codons in order.
  • If the codon list ends before derivation finishes, wrap back to the start.
  • For a fixed grammar, the mapping is deterministic.

Worked Derivation Pattern

The notes highlight this as the main practical section to study carefully.

Example setup Codon decimals: 5, 166, 4, 30, 172, 2
<S> ::= <X><W><W> | <W><X><Y> | <X><W><Y> <W> ::= <Y><X> | <Y><Z> <X> ::= 1|2|3|4|5|6|7|8|9|0 <Y> ::= a|b|c|d|e|f|g|h|i|j <Z> ::= @|~|#|&|%|*|)|-|(|+
1
Expand <S>.

There are 3 rules, so 5 % 3 = 2. Choose rule 2: <X><W><Y>.

2
Expand the leftmost <X>.

There are 10 rules, so 166 % 10 = 6. Choose the symbol 7.

3
Expand <W>.

There are 2 rules, so 4 % 2 = 0. Choose <Y><X>.

4
Continue left to right.

At each step, use the next codon and the same modulo rule against the current non-terminal's rule count.

Tutorial-Style Boolean Mapping Example

Suppose the grammar is:

expr ::= expr op expr | term op ::= AND | OR | XOR | NAND term ::= NOT var | var var ::= A | B | C | 0 | 1

And the codon sequence is [1101] [1010] [0111], which in decimal is 13, 10, 7.

1
Start with expr.

There are 2 rules, so 13 % 2 = 1. Select term.

2
Expand term.

There are 2 rules, so 10 % 2 = 0. Select NOT var.

3
Expand var.

There are 5 rules, so 7 % 5 = 2. Select C.

Final phenotype: NOT C

Why Wrapping And Introns Matter

Wrapping If the derivation is not finished when the codon list ends, the mapping process loops back to the start of the genome and continues consuming codons.
Introns If derivation finishes before all genetic material is used, the unused part is effectively inactive for that phenotype. Those unused pieces are often called introns.
In a worked answer, it helps to state explicitly whether all codons were used, whether wrapping occurred, and whether unused codons remain.

What To Do In A GE Exam Question

  • Write the grammar and count the number of rules for the current non-terminal.
  • Convert each codon to decimal if the question gives binary codons.
  • Apply the modulo rule exactly and show the remainder each time.
  • Always expand the leftmost non-terminal.
  • Track wrapping if the codon sequence runs out before the phenotype is complete.
  • Finish by stating the phenotype or derived final expression clearly.

Exam Checklist

  • Define genotype, codon, phenotype, and grammar.
  • Write and use the modulo mapping equation correctly.
  • Explain leftmost derivation and wrapping.
  • Describe why GE uses single-point crossover on variable-length genomes.
  • Work a derivation trace step by step.
